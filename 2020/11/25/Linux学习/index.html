2. 入门认识

1. 系统分区

  1. 主分区：最多四个
  2. 扩展分区
    1. 最多只能有一个
    2. 由主分区生成
    3. 不能写入数据，只能包含逻辑分区
    4. 逻辑分区的编号永远都是从5开始，因为不能占用前四个
  3. 逻辑分区：可以正常的格式化和存储数据
  4. 必须分区
    1. /：根分区，所有分区都是根分区的子目录
    2. 每个子目录也可以划分自己的硬盘空间
    3. swap分区：相当于虚拟内存
  5. 推荐分区
    1. /boot分区：保证在根分区写满的时候，linux还能启动，一定是sda1

格式化的目的：为了在硬盘当中写入文件系统（block）

格式化会将硬盘分成一个一个等大小的数据块，默认4kb，但是存储文件并不是连续的存储，而是散乱的。

磁盘碎片整理：将散乱的存储尽量的连续整齐起来，使得有更大，更好的空间可以使用。

文件的分区表：每个文件都有编号（INode号），通过查表得到编号然后去磁盘读取

硬盘设备文件名：/dev/硬件的设备名

1. SATA硬盘：/dev/sd[a-d]
 2. 光驱：/dev/sr0 或/dev/cdrom
 3. 鼠标：/dev/mouse
 4. 网卡：eth0
 5. 在linux中，所有硬件都是文件

==Linux：分区，格式化，给分区加入设备文件名，挂载（分配盘符）==

Mounted：挂载。mount：准备，安排

4. 网络

  1. 桥接：虚拟机利用真实网卡与真实计算机进行通信
    1. 好处：配置简单，设置同一个网段的ip可以直接访问
    2. 坏处：需要占用网段的一个ip
  2. NET：通过虚拟网卡通讯，只能和你的电脑通讯，但是不能和局域网通讯
  3. Host Only：只能和本机通讯，不能访问互联网

ifconfig：查看网卡信息

  1. lo网卡：loop back，本机回环网卡，主要是本机通讯的测试。作用是不需要插网线，也可以ping同自己

  2. eth0：虚拟机的网卡

5. 初学建议

  1. Linux严格区分大小写

  2. 在Linux里面，所有东西都是通过文件保存的。在Linux中修改的配置如果不写入文件，那就仅仅是暂时生效，一旦重启就不起作用。

  3. Linux没有扩展名，Linux不靠扩展名来区分文件，==而是靠权限==。但是还是可以加上扩展名，便于使用

    输入 ls /bin/ 来查看可执行程序。

  4. Linux的所有存储设备都必须挂载之后用户才能使用

    1. 挂载：把设备文件名和挂载点连起来的过程就叫挂载
    2. 在windows上，一切盘符，U盘，都是自动挂载
    3. 在Linux，在分区和硬盘是自动挂载，但是U盘之类的，插入之后还得手工挂载（分配盘符）

1. 各个目录介绍

bin（二进制文件，二进制可执行文件在Linux里就是命令）

  1. /bin/：存放系统命令，普通用户和超级用户都可以执行
  2. /sbin/：存放和系统环境设置相关的命令，只有超级用户可以使用
  3. usr/bin/：存放系统命令，谁都能执行（单用户模式下不能执行）
  4. usr/sbin/：存放和文件系统不必要的系统管理命令，只有超级用户可以使用（单用户模式下不能执行）
  5. /boot/：系统启动目录，保存系统启动的相关文件
  6. /dev/：设备文件保存位置
  7. /etc/：专门的默认的Linux的配置文件保存位置
  8. /home/：普通用户的家目录，管理员的家目录：/root
  9. /lib：系统调用的函数库
  10. /lost+fount/：系统意外奔溃，产生一些文件碎片放在这里。fsck会检查这里，并且修复
  11. /media/：挂载目录
  12. /mut/：挂载目录
  13. /misc/：挂载目录
  14. /opt/：第三方安装的软件
  15. /proc/：虚拟文件系统
  16. /sys/虚拟文件系统
  17. /root/：超级用户的家目录
  18. /srv/：服务数据目录
  19. /tmp/：零时文件
  20. /usr/：（Unix Software Resource）系统资源目录
  21. /var/：动态数据保存位置

  4. 在配置防火墙时，不要把自己踢出服务器

  5. 指定合理的密码
  6. 分配合理的权限
  7. 定期备份重要数据和日志

  1. 个别命令不遵循此格式

1. 文件操作命令

1. ls

  1. 英文原意：list
  2. 所在路径：/bin/li（所有用户都能执行）
  3. 参数
    1. -a：显示所有文件，隐藏文件以. 开头。隐藏文件：这是系统文件，没事别乱动
    2. 路径：参看指定目录下的文件
    3. -l（long）：长格式显示，显示相关属性
      1. 第一个root：所有者，可以转移
      2. 第二个root：所属组，定义什么组可以操作
      3. d：开头代表目录
      4. l：开头代表软连接
      5. -：开头代表文件
      6. 除了第一个字母以外，后面9个字母，3个一组，分别对应所有者，所属组，其他人的权限
    4. -h（human）：人性化显示
    5. -d：经常与l一起使用，只显示指定目录的信息，而不看数据
    6. -i（iNode）：查看id号

2. mkdir

  1. 英文原意：make directories

    1. -p：递归创建，可以创建多级文件夹

1. 进程管理

进程是一个正在运行中的程序，ls命令也会产生进程

  • 判断服务器的健康状态
  • 查看系统中的所有进程

1. 进程查看

==查看所有进程：ps==

  1. ps aux
    1. a：前台进程
    2. x：后台进程
    3. u：查看什么用户产生的
  2. ps -le
    1. l：更见详细的信息
    2. e：查看所有进程

ps aux：显示解析

  1. USER ：用户
  2. PID：线程号
  3. %CPU ：cpu占比
  4. %MEM：内存占比
  5. VSZ：占用虚拟内存大小，单位kb
  6. RSS：占用实际内存大小，单位kb
  7. TTY：由那个终端产生
  8. STAT：状态
    1. R：运行
    2. S：休眠
    3. T：停止
    4. s：包含子进程
    5. +：位于后台
  9. START：启动时间
  10. TIME：占用CPU运算时间
  11. COMMAND：名字

==查看系统健康状态：top==

top [选项]

选项

  1. -d：秒数，默认3秒

输出解析：头五行

第一行：服务器信息

第二行：进程信息

第三行：CPU信息，98.3 id表示空闲比

第四行：真实物理内存信息

第五行：交换分区信息

==查看进程树：pstree==

-p：详细信息

-u：显示所属用户

pkill [选项] [性号] 进程名

2. 工作管理

1. 把进程放入后台

  1. 在命令最后加入 &

  2. 在执行命令中，使用 CTRL + Z，命令暂停

查看后台工作：jobs -l

将后台工作恢复到前台：fg %1

将后台工作恢复到后台：bg %1

3. 系统资源查看

  • 监控系统资源：vmstat
  • 显示开机时内核自检信息：dmesg
  • 查看内存使用状态：free -m
    • 缓冲：将输出数据放在缓冲中，写入效率将更快
    • 缓存：将硬盘的数据存在缓存中，读取数据将更快
  • 查看cpu信息：cat /proc/cpuinfo
  • 查看系统与内核相关信息：uname [选项]
    • -a：所有信息
    • -r：查看内核版本
    • -s：查看内核名字
  • 查看系统位数：file /bin/ls
  • 列出进程调用的文件
    • -c 字符串
    • -u 用户名
    • -p pid

4. 系统定时任务

service crond restart，cron是定时任务的程序，d是守护进程的意思

使用：crontab [选项] ，来设置定时任务

-e：编辑

-l：查询

-r：删除当前用户所有的定时任务

crontab -e，打开编辑器。标准格式

五个 * 表示时间

  1. 一个小时当中的==第==几分钟
  2. 一天当中的==第==几个小时
  3. 一个月当中的==第==几天
  4. 一年当中的==第==几个月
  5. 一周当中的星期几

特殊符号

  1. *：任何时间
  2. ,：不连续的时间
  3. -：连续的时间
  4. */n：每隔多久执行一次
